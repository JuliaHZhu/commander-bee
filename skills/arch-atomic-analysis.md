# 原子化文本分析

把任意长度的文本切成原子笔记、打标签、看统计。**分析结果全部存盘，不在对话里堆 token**。

## 存储位置

```
~/.worker-bee/arch/texts/<name>/
├── source.md      ← 原始文本（URL 提取或用户粘贴）
├── atoms.md       ← 原子序列（每个原子一行：ID + 标签 + 字数 + 首句）
├── stats.md       ← 章节级统计表 + 整体数据特征
└── template.md    ← PM Bee 填空模板（可选，从 stats 生成）
```

`<name>` = 文章标题或用户指定的英文短名（如 `three-body-ch3`）。

## Phase 1: 切原子

**什么时候**：用户说 "分析这篇" / "切一下" / "原子分析"

**步骤**：
1. 拿到文本（URL → `net_web_extract`，或用户粘贴 → 直接写 `source.md`）
2. 按体裁切原子：

| 体裁 | 原子定义 | 示例 |
|------|---------|------|
| 小说 | 同一场景内连续对话 = 1原子；场景切换/视角切换/时间跳跃 = 新原子 | "他说……她应……两人沉默。" = 1原子 |
| 论文 | 一个自然段 = 1原子 | 按原段落切 |
| 申论 | 一个自然段 = 1原子 | 按原段落切 |

3. 写入 `atoms.md`，格式：

```
# Atoms: <name>
## 体裁: 小说 | 论文 | 申论
## 总原子数: (切完后更新)

### Chunk 1: <章节名或分段标记>

#001 [标签待打] [字数]
我去买几个橘子。你就站在此地，不要走动。
#002 [标签待打] [字数]
他望车外看了看，说："我买几个橘子去。"
```

**约束**：
- 每批最多切 500 原子，超过时分多次切（"第1-500原子切完，继续？"）
- 原子 ID 三位数补零（`#001`），方便排序
- 字数 = 中文字符数（用 `chars`，不用 tokens）
- 原子原文必须保留——存档用，不是给对话看的

## Phase 2: 打标签

**什么时候**：用户说 "打标签" / "标一下"；或 Phase 1 结束后自然接

**标签体系**：

| 体裁 | 标签 | 含义 |
|------|------|------|
| 小说 | `对话` | 人物对话（含引述句） |
| 小说 | `叙述` | 叙事推进、背景交代 |
| 小说 | `描写` | 环境/外貌/感官描写 |
| 小说 | `动作` | 人物动作/行为 |
| 小说 | `心理` | 内心活动/独白 |
| 小说 | `转场` | 场景切换/时间跳跃标记 |
| 论文 | `立论` | 提出论点 |
| 论文 | `说理` | 论证/推理 |
| 论文 | `举例` | 案例/数据/实证 |
| 论文 | `引用` | 引用他人研究 |
| 论文 | `过渡` | 段落间衔接 |
| 论文 | `总结` | 章节收束 |
| 申论 | `破题` | 问题界定 |
| 申论 | `分析` | 原因/影响分析 |
| 申论 | `对策` | 解决方案 |
| 申论 | `收束` | 总结升华 |
| 申论 | `引用` | 政策/理论引用 |

**步骤**：
1. 读 `atoms.md`，每次读 200 原子一批
2. 逐个打标签（1-3 个标签，不打多）
3. 更新 `atoms.md`，把 `[标签待打]` 替换为 `[对话]` 等
4. 标签全部打完 → 更新文件头 `总原子数: N` + `标签完成: 是`

**约束**：
- 宁可漏打，不要强打——拿不准的标 `[其他]`
- 原子原文不删不改
- 多次打标签时，跳过已打完的（`[标签待打]` 以外的）

## Phase 3: 看统计

**什么时候**：标签全部打完；或用户说 "统计" / "数据特征"

**步骤**：
1. 读 `atoms.md`，逐原子扫标签和字数
2. 按 Chunk 分组统计 → 写入 `stats.md`

**stats.md 格式**：

```markdown
# 统计: <name>

## 整体
- 总原子数: 847
- 总字数: 124,500
- 平均原子长: 147字

## 标签分布
| 标签 | 原子数 | 占比 | 平均字数 |
|------|--------|------|---------|
| 对话 | 423 | 49.9% | 68 |
| 叙述 | 215 | 25.4% | 210 |
| 描写 | 89 | 10.5% | 185 |
| 动作 | 72 | 8.5% | 120 |
| 心理 | 35 | 4.1% | 240 |
| 转场 | 13 | 1.5% | 45 |

## 章节对比
| 章节 | 原子数 | 对话% | 叙述% | 描写% | 其他% |
|------|--------|-------|-------|-------|-------|
| 第一章 | 124 | 55 | 20 | 12 | 13 |
| 第二章 | 98 | 42 | 30 | 15 | 13 |
| 第三章 | 156 | 68 | 15 | 8 | 9 |
| ... | ... | ... | ... | ... | ... |

## 特征
- 对话占主导（50%），但第三章异常高（68%）
- 描写集中在开头章节（第一、二章描写占比 > 10%）
- 转场极少（1.5%），场景切换频率低
- 心理描写最后三章归零
```

**一句话特征**（放最顶上，Architecture Bee 对话入口）：

> 对话驱动型叙事，第三章对话密度异常。描写集中在开头，后期偏动作推进。场景切换少（线性时间线）。

## Phase 4: 生成模板（给 PM Bee）

**什么时候**：统计完成 + 用户说 "生成模板" / "PM用"

**步骤**：
1. 从 `stats.md` 取章节级数据
2. 生成 `template.md`

```markdown
# PM Template: <name>

## 目标
仿写一篇 [体裁] 风格文章，模仿原文结构。

## 各章任务
| 章节 | 原子数 | 对话占比 | 任务 |
|------|--------|---------|------|
| 第一章 | 124 | 55% | 对话55原子/叙述25/描写15/动作12/其他17 |
| 第二章 | 98 | 42% | 对话41原子/叙述29/描写15/动作8/其他5 |
| ... | ... | ... | ... |

## 章节模板: 第一章
- 开头: 叙述 3-4原子（引入场景）
- 中段: 对话为主 50-60原子，穿插动作
- 结尾: 描写 5-8原子（收束情绪）
- 关键: 描写占比 > 10%，心理 > 5%
```

## 跨体裁通用规则

- 原子不嵌套——每段原文只属于一个原子
- 原子可合并可拆分——对话里夹一句描写，归对话（看主功能）
- 标签打**主功能**——一段里 80% 对话 + 20% 动作 = 标 `对话`
- Chunk = 章/节/分段标记，用户提供或自动识别（`##` / `第X章` / 空行分隔）
- Phase 1/2/3 可以跨多轮对话——`atoms.md` 里 `标签完成: 否` 就是还没打完
