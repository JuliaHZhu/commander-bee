# 小说引擎标签系统

双模式，由 novel-split 报告的对白占比自动选择：

- **道尔模式**（对话≤50%）: 10 基础标签。小观察点 atom。
- **克里斯蒂模式**（对话>50%）: 14 标签。大对白回合 atom——矛盾/表演/审讯/留白在一个 atom 内可见。

不再需要关系层。克里斯蒂模式下，atom 本身就是交换单元。

## 存储位置

```
~/.worker-bee/arch/texts/<作品名>/<段-slug>/
├── source.md          ← 原文（只读）
└── atoms.md           ← 输入: [待打] → 输出: 原地更新
```

---

## 道尔模式 — 10 标签（对话≤50%）

Watson 在"看"。Atom 是小观察点。每原子 1-3 标签。

| 标签 | 含义 | 判断标准 |
|------|------|---------|
| **线索** | 指向真相的信息碎片 | 删掉后读者少一条推理路径 |
| **误导** | 故意引向错误方向 | 删掉后读者不会被骗 |
| **张力** | 紧迫感/威胁升级 | 时间压力、威胁逼近 |
| **揭示** | 部分真相浮出 | "原来如此" |
| **动机** | 为什么这么做（why） | 行为心理原因 |
| **手法** | 怎么做到的（how） | 犯罪手法暗示 |
| **人物** | 人物建立/关系 | 纯角色刻画 |
| **场景** | 环境/氛围 | 纯空间描写 |
| **转折** | 预期被推翻 | 揭示拼图，转折掀桌 |
| **收束** | 解谜/填坑 | 伏笔兑现 |

## 克里斯蒂模式 — 14 标签（对话>50%）

Atom 是**对白回合**。一个 atom 内有问→答→回避→矛盾→追问的完整动态。

### 基础 10（同上）照常标。

### 扩展 +4（在对白回合 atom 内可见）

| 标签 | 含义 | 为什么在对白回合内可见 |
|------|------|----------------------|
| **矛盾** | 两个说法/视角冲突 | Atom 内包含双方：A 说 X，B 说 Y 与之冲突。冲突在 atom 里 |
| **留白** | 该说没说、回避 | Atom 内有提问 + 非回答。空隙在 atom 里 |
| **表演** | 角色扮演/伪装 | Atom 捕捉完整表演：铺垫→伪装→套话 |
| **审讯** | 问答即推理 | Atom 就是问答回合。谁问谁答谁回避——都在一个 atom |

### 标注示例

```
汤米装作收藏家接近嫌疑人，闲聊中套出画作来源。
→ [表演] [审讯] [线索]
表演: 整段在演收藏家
审讯: 问答结构（问→恭维→套出）
线索: 画商名字到手
```

```
"那天晚上我一直在家。"
"房东太太说你九点才回来。"
"她一定记错日子了。"
探长低头记着什么，没抬眼。
→ [矛盾] [张力]
矛盾: "在家" vs "九点回来"——冲突在 atom 内
张力: 探长记笔记但不说话——压力在升级
```

```
"周二你在哪？"
"墙上那幅画真漂亮。是特纳的吗？"
"我问的是周二。"
"我一直喜欢艺术。我母亲是画家……"
警探等着。
→ [留白] [表演]
留白: 三个回答，没一个答问题
表演: 扮艺术爱好者转移话题
```

### 互斥规则

| 对 | 规则 |
|----|------|
| `线索` vs `误导` | 真相还是障眼法？二选一 |
| `揭示` vs `转折` | 拼图还是掀桌？二选一 |
| `矛盾` vs `误导` | 冲突还是故意骗？通常二选一 |
| `表演` vs `人物` | 演戏还是真实？二选一 |
| `审讯` vs `线索` | 过程还是结果？可共存 |

## 打标签流程

1. 看 atoms.md 头或 novel-split 报告——对话>50%？→ 切换模式
2. 读 atoms.md，找 `[待打]`，每批 200
3. 读原文 → 对照 Deck → 挑 1-3 标签
4. 更新 + 标记完成
