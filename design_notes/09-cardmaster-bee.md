# 09 — CardMasterBee

> 独立软件——万物卡牌化引擎。不是 Skill，是独立程序暴露 Tool 接口。所有内容/概念/决策/方法都可简化为卡牌，成为标准化操作单元。

---

## 一、什么是卡牌

卡牌是标准化信息原子，包含：

```
┌──────────────────────┐
│ 卡牌类型: 概念卡       │
│ 来源: Hayes Ch3       │
│ 章节: 第3章           │
│ 题面: Bootstrap CI 和 │
│   Sobel test 的核心区 │
│   别是什么？          │
│ 解答: Bootstrap CI    │
│   不做正态假设...     │
│ 难度: ★★☆☆☆          │
│ 上次答对: 2026-06-21  │
│ 间隔: 2天             │
│ 标签: #方法 #中介     │
└──────────────────────┘
```

---

## 二、卡牌类型

| 类型 | 内容 | 来源 |
|------|------|------|
| **概念卡** | 一个术语/一个知识点 | Hayes教材、教科书 |
| **判断卡** | 一个决策/一条经验 | 战略案例、塔勒布 |
| **论证卡** | 一个论证链/一个逻辑推理 | 归谬法输出 |
| **方法卡** | 一个操作步骤/一个流程 | PROCESS教程 |
| **数据卡** | 一个关键数字/一个事实 | 西门子年报 |
| **问题卡** | 一个待验证的问题 | 博士论文 |
| **观点卡** | 一个独立判断/一句话 | 阅读笔记 |

---

## 三、为什么卡牌化

**对StrategicBee**：标准化操作单元。战略分析不是"写一篇分析"，而是"从卡牌库中抽取相关卡→用五工具流水线排列组合→产出一张新的判断卡"。

**对球机**：球机抛的不再是"任务"，而是卡牌。"今天复习哪几张概念卡？""今天解哪几张问题卡？"

**对LLM强化学习**：每张卡牌=一个训练样本。题面=输入，解答=标签。每次答题反馈一个reward信号。难度动态调整。间隔重复算法（类似Anki的SM-2但用LLM判断难度）。

**对操作员**：知识不再散落在一百多篇论文和课程里。所有的东西都可以被抽出来、洗牌、组合、重排。

## 四、卡牌生命周期

```
文本/概念/判断
    │
    v
┌─────────────┐
│ 提取 → 卡牌化 │  ← 半自动：LLM提取 + 人工确认
└──────┬──────┘
       v
┌─────────────┐
│ 存储 → 卡牌库 │  ← Markdown或SQLite
└──────┬──────┘
       v
┌─────────────┐
│ 分发 → 流转   │  ← 球机（今天抛哪张？）
│              │  ← 刷题页（检验掌握度）
│              │  ← StrategicBee（组合分析）
└──────┬──────┘
       v
┌─────────────┐
│ 反馈 → 更新   │  ← 难度调整
│              │  ← 间隔更新
│              │  ← 关联图谱更新
└──────┬──────┘
       v
┌─────────────┐
│ 退役 → 归档   │  ← 连续答对 N 次 → 进入长期记忆库
└─────────────┘
```

## 五、与现有系统的接口

| 现有组件 | 卡牌化接口 |
|---------|-----------|
| **Hayes刷题页** | 已经是CardMasterBee的最简原型——97张概念卡 |
| **量表汇总笔记** | 每一条=一张方法卡 |
| **价值汇总笔记** | 每一条=一张观点卡 |
| **球机** | 球机画球→抽卡→答题→更新间隔 |
| **session handoff** | 上下文中携带当前卡组状态 |

## 六、强化学习循环

```
操作员打开刷题页
    │
    v
球机抽取卡牌（考虑难度+间隔+上次结果）
    │
    v
操作员答题（对/错）
    │
    v
┌──────────────────┐
│ Reward 信号:      │
│ ✓ 对 → 间隔延⻓     │
│ ✗ 错 → 间隔缩短    │
│ ⏱ 反应时间 → 难度调 │
└──────┬───────────┘
       v
更新卡牌参数 → 下一轮
```

LLM在这个循环中的角色：不只是出题——它可以根据操作员的错误模式**生成针对性的新卡牌**。如果你连续错了3张关于"间接效应CI解读"的卡，LLM应该生成第4张来专门打这个弱点。

## 七、卡牌存储格式

建议用Markdown frontmatter（和job存储同一哲学）：

```markdown
---
id: CARD-042
type: concept
source: Hayes Ch3
chapter: 第3章
question: Bootstrap CI 和 Sobel test 的核心区别是什么？
answer: Bootstrap CI 不对间接效应的抽样分布做正态假设，Sobel test 假设 a*b 服从正态分布。
difficulty: 2
last_correct: 2026-06-21
interval: 2d
consecutive_correct: 3
tags: [方法, 中介, Bootstrap]
---
```
