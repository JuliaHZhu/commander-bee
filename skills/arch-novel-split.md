# 小说原子切割

无交互，全自动。小说文本 → 按场景×主题切原子 → 写 atoms.md。

## 存储位置

arch-novel-split 在**当前目录**跑——读该目录下的 `source.md`，写 `atoms.md`。

**独立长篇**（没跑 arch-collection-split）：
```
~/.worker-bee/arch/texts/<作品名>/
├── source.md      ← 把原始文本放这里
└── atoms.md       ← 产出
```

**流水线中**（arch-collection-split 之后，在子目录跑）：
```
~/.worker-bee/arch/texts/<作品名>/
├── source.md              ← 合集原始文本
├── index.md
├── <段-slug>/
│   ├── source.md          ← arch-collection-split 写的
│   └── atoms.md           ← arch-novel-split 写到这里
└── ...
```

## 切割规则

### 一级边界：场景切换（必切）

以下任一发生 = 新原子：
- 场景/地点变换
- 视角切换（A→B）
- 时间跳跃（"三天后"）
- 章节分段（第X章）

### 二级边界：同场景内按主题切

**对话**：
- 同场景连续对话 → 按**主题内容**切，不按"谁在说话"切
- 主角揭露动机 → 1个原子
- 主角接着揭露手法 → 另1个原子
- 两人斗嘴/闲聊（功能单纯） → 1个原子
- 原则：一个原子 = 一个可识别的信息单元

**描写**：
- 按描写对象切：写环境 → 1原子，写人物外貌 → 1原子
- 同一对象的连续描写不切

**动作**：
- 一个连续动作序列 = 1原子
- 动作类型切换（追逐→对峙→逃脱）= 各自成原子

**心理**：
- 一段连续内心活动 = 1原子
- 思绪跳跃/话题转换 = 新原子

### 三级标注：切的同时打表面标记

每个原子在切时就标上表面类型：

| 标记 | 含义 |
|------|------|
| `[对话]` | 人物对话（含引述句） |
| `[描写]` | 环境/外貌/感官 |
| `[动作]` | 人物行为/事件推进 |
| `[心理]` | 内心独白/意识流 |
| `[混合]` | 对话中夹大量描写/动作，无法分清主次 |

可选标注（有把握才标）：
- `[对话-动机]` / `[对话-手法]` / `[对话-关系]` — 对话主题
- `[描写-场景]` / `[描写-人物]` / `[描写-氛围]` — 描写对象
- `[动作-追逐]` / `[动作-对峙]` — 动作类型

## 输出格式

atoms.md：

```markdown
# Atoms: <name>
## 体裁: 小说
## 总原子数: <N>
## 标签完成: 否

## 第一章: <章节名>

#001 [待打] [180字] [描写-场景]
胡同里飘着煤烟味。路灯把雪地照得发黄。
老张缩在军大衣里，眼睛盯着对面的单元门。

#002 [待打] [340字] [对话]
"你等了多久了？"
"三个钟头。他没出来过。"
"不可能。后门呢？"
"堵死的。"
老张把烟头捻灭在墙上，火星溅了他一手。他没躲。
"你确定他是七点进去的？"
"我看着他进去的。"

#003 [待打] [120字] [动作]
老张突然站起来。大衣掉在地上。
他没捡。直接朝单元门走去。
```

## 完成后报告

```
✓ <name>
  体裁: 小说
  原子数: 847
  总字数: 124,500
  平均原子长: 147字
  
  表面标记分布:
  对话 55% / 描写 22% / 动作 15% / 心理 6% / 混合 2%
  
  → 标签模式: 对白密集型（对话>50% — arch-novel-tag 将激活扩展4标签）
  → 下一步: "打标签"
```

对话 ≤50% 时：
```
  → 标签模式: 叙事驱动型（arch-novel-tag 仅用基础10标签）
```

## 约束

- 不提问——体裁已固定为小说，不回头确认
- 每篇切完立即写 atoms.md
- 超过 2000 原子时分段写，不攒内存
- 原子原文完整保留，不删改
- 表面标记只是辅助——真正的引擎标签在 Phase 2（arch-novel-tag）
